# 🐺 Читавук

**Читавук** (от рус. *читать* + серб. *вук* — волк) — приложение для изучения сербского языка на основе чтения текстов.
Система обеспечивает глубокую интеграцию NLP-инструментов, позволяя проводить контекстный морфологический разбор сербских слов непосредственно при чтении PDF и DOCX форматов. Реализовано сохранение лексики в колоды для интервального повторения.

## Архитектура и основные возможности

- **Анализ текста в реальном времени:** Извлечение токенов по нажатию, определение части речи (UPOS), лемматизация и контекстный перевод. Поддерживается автоматическая коррекция поврежденных диакритических знаков сербской латиницы (š/č/ć/ž/đ).
- **Грамматический движок:** Автоматическая генерация парадигм склонения (7 падежей) и спряжения (prezent/perfekat/futur). Разбор синтаксических ролей.
- **Модуль библиотеки:** Парсинг PDF (через `syncfusion`) и DOCX. Интегрированная локальная БД (SQLite) для управления коллекциями и сохранения прогресса чтения.
- **Интервальное повторение (SRS):** Реализация алгоритма, основанного на SM-2, для закрепления словаря.
- **Бесплатная библиотека:** Встроенная директория `assets/library/` для предзагрузки базовых материалов в бандл без необходимости скачивания пользователем.

## Стек технологий

- **Клиентская часть:** Flutter (Android/Windows Desktop), `provider` (state management), `sqflite` (локальное хранилище), `syncfusion_flutter_pdf` (рендеринг документов).
- **Серверная часть:** Python 3.10, FastAPI, `classla` (NLP-модель, адаптированная для южнославянских языков). Сервер работает полностью автономно, имеет встроенный daemon-поток (keep-alive) для обхода ограничений "засыпания" бесплатных tier-ов.
- **Лексическая база:** Офлайн-словарь (~21k форм), сгенерированный скриптами из открытого трибанка `UD_Serbian-SET`.

## Сборка и развертывание

### Клиентское приложение (Android / Windows)
Сборка для Android автоматизирована с помощью **GitHub Actions**.
При пуше в репозиторий может быть вручную запущен CI-пайплайн (см. `.github/workflows/build-apk.yml`), который компилирует `app-release.apk` и прикрепляет его в артефакты сборки. Это исключает необходимость установки локального Android SDK на рабочей станции.

Сборка для Windows Desktop:
```bash
cd frontend
flutter clean
flutter pub get
flutter build windows --release
```

### Развертывание бэкенда (Hugging Face Spaces / Docker)
Бэкенд подготовлен для работы в контейнеризированной среде на мощностях Hugging Face Spaces.
В корне проекта лежит оптимизированный `Dockerfile` (базовый образ `python:3.10-slim`). Код выполняется непривилегированным пользователем (UID 1000).
Для деплоя достаточно привязать репозиторий к HF Space и выбрать Docker SDK. Запуск сервера осуществляется на стандартном для платформы порту `7860`.

*Примечание: клиент может функционировать без выделенного сервера (в offline/direct-translate режиме), однако точность контекстного снятия омонимии обеспечивается только при интеграции с NLP-бэкендом.*
